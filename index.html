About

Builder. Leader.
Identity Architect.

23 years across fintech, telecom, SaaS, and cloud-native platforms.

I spent five years at Aquera Labs architecting and owning an enterprise IAM orchestration platform — a multi-tenant, fault-tolerant system processing 10M+ identity records monthly across 5,000+ concurrent integration jobs. I did not just lead this platform — I coded it daily.

I hand-built a SCIM 2.0 filter parser from the RFC 7644 specification, implementing full AST construction and interpretation. I designed distributed coordination primitives using Redis — leader election, distributed mutex, pub/sub cancellation, heartbeat-based zombie detection. I shipped production AI features using AWS Bedrock and RAG.

Before Aquera: I architected a telecom platform for 30M+ users at SinarMas, led engineering for core banking used by 300+ banks at FIS, and built teams of 8–25 engineers across Capgemini, KPIT, CGI, and IBM.

I hold an M.Tech from NIT Calicut and am based in Bangalore, India — open to hybrid and remote EM and Principal Architect roles in IAM, platform engineering, and cloud-native SaaS.

⚡ Generative AI · Production

Aquera Assistant — RAG-based Integration Code Generator

A production Retrieval-Augmented Generation system that generates executable SCIM integration code from natural language. Built on AWS Bedrock — dynamically retrieves source and destination application metadata, injects it as context into the LLM prompt, and returns ready-to-run integration code. Enterprise users type "Get all active users from Okta" and receive executable code automatically.

AWS Bedrock RAG Node.js SCIM 2.0 DynamoDB REST API
  • Dynamic metadata retrieval — source and destination app context injected at inference time
  • Natural language to executable SCIM code — no manual adapter writing required
  • Shipped to production, serving Fortune-class enterprise customers
  • Reduces integration time from days to minutes for supported connectors
🧠 ML Integration · Production

Identity Analytics ML Pipeline

Async ML pipeline for enterprise identity analytics. Identity Explorer Engine transforms SCIM user data into structured CSV, uploads to S3, and coordinates with an ML engine for Group Rules Analysis and OU Analysis via heartbeat-based async coordination.

  • SCIM → CSV → S3 → ML engine data pipeline
  • Heartbeat coordination — 60-second keep-alive prevents timeout on large datasets
  • Group Rules Analysis and OU Analysis surfaced to UI
  • Handles enterprise-scale user datasets reliably
Node.js AWS S3 SCIM ML Engine Heartbeat Protocol
